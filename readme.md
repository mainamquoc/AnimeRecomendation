# Anime Recommendation System

A collaborative-filtering anime recommender built with the MyAnimeList anime-ratings dataset. The project cleans explicit user ratings, explores the data, and trains/evaluates a Singular Value Decomposition (SVD) model to predict ratings and produce top-10 recommendations.

## Project structure

```text
datasets/              Raw source data (`anime.csv`, `rating.csv`)
cleaned_data/          Cleaned datasets generated by notebook 01
notebooks/
  01_eda_data_preparation.ipynb
  02_svd_model_evaluation.ipynb
outputs/               Evaluation metrics, recommendations, and figures
Latex-report/       LaTeX source and compiled project report
```

## Dataset

The data contains ratings from 73,516 users across 12,294 anime.

- `datasets/anime.csv` - anime metadata: title, genres, format, episodes, community rating, and membership count.
- `datasets/rating.csv` - user-anime interactions on a 1-10 scale. A rating of `-1` means watched but not explicitly rated.

See [datasets/about_datasets.md](datasets/about_datasets.md) for the source dataset description and acknowledgement of the MyAnimeList API.

## Workflow

1. Run `notebooks/01_eda_data_preparation.ipynb` from the project root.
   - Validates input schemas and profiles data quality.
   - Removes missing, invalid, and `-1` (unrated) interactions.
   - Merges duplicate user-anime pairs by mean rating and deduplicates metadata.
   - Writes `cleaned_data/ratings_clean.csv` and `cleaned_data/anime_clean.csv`.
   - Produces exploratory figures in `outputs/figures/`.
2. Run `notebooks/02_svd_model_evaluation.ipynb`.
   - Samples up to 1,000,000 ratings and applies iterative 5-core filtering to users and anime.
   - Uses a reproducible 80/20 outer holdout split (`seed=42`).
   - Performs lightweight validation tuning over SVD latent factors (30, 50, and 100), then fits the selected model.
   - Exports metrics, prediction diagnostics, and personalized top-10 recommendations.

## Setup and run

Use Python 3.10+ and install the notebook dependencies:

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows PowerShell
pip install jupyter pandas numpy matplotlib seaborn scikit-surprise
jupyter notebook
```

Open and run the notebooks in their numbered order. They resolve paths correctly when launched either from the project root or from `notebooks/`.

> `scikit-surprise` may require a C/C++ build toolchain on some Windows Python installations. If it is missing when the model notebook starts, the notebook attempts to install it with `%pip`.

## Current model results

The latest run uses an SVD model with 30 factors, 20 epochs, learning rate `0.005`, regularization `0.02`, and `seed=42`.

| Metric | Result |
| --- | ---: |
| RMSE | 1.2551 |
| MAE | 0.9560 |
| Precision@10 | 0.6397 |
| Recall@10 | 0.8273 |

These metrics use the outer holdout split over 952,497 filtered ratings (190,500 predictions). Ranking metrics use positive ratings of 8 or above and evaluate eligible users with at least 10 held-out items.

## Generated outputs

- `outputs/model_evaluation_metrics.csv` - RMSE, MAE, Precision@10, and Recall@10.
- `outputs/top_10_recommendations.csv` - ranked recommendations with anime metadata and predicted ratings.
- `outputs/figures/` - rating distributions, user activity, error distributions, error/ranking metrics, and actual-vs-predicted plots.
